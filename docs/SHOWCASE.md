# Showcase 说明

这个仓库主要是拿来讲低代码的，不是拿来当完整产品直接交付的。

如果只是想给别人说明两件事：

- 前端低代码一般怎么搭页面
- 后端低代码一般怎么配模型、流程和动作

那这套演示已经够用了。

## 前端部分在看什么

前端这边重点放在页面搭建。

能看到的主要就是这些内容：

- 左边是组件库
- 中间是画布
- 右边是属性面板
- 拖进去的组件可以改文案、改字段、调顺序

它更像活动页搭建器、表单搭建器、运营页面编辑器。

## 后端部分在看什么

后端这边重点不是页面，而是业务逻辑配置。

这里拆了几块常见能力：

- 数据模型：字段、类型、规则
- 流程编排：节点、角色、条件
- 动作链：通知、落库、系统同步
- 接口编排：一个请求进来之后要走哪些处理步骤

这部分更像流程引擎、集成平台或者业务后台编排器。

## 配完之后通常怎么落地

这类东西一般有两条路：

1. 运行时解释

把配置存下来，运行时引擎按配置去渲染页面、执行流程、调用动作。

2. 生成代码或配置

把页面、模型、流程和接口编排转成项目代码、表结构、流程定义或者 DSL。

## 这个仓库现在做到哪一步

现在这版更偏“展示”和“讲解”。

它已经把设计器最关键的外观和交互做出来了，适合拿来解释概念、做演示、录视频，或者继续往下扩成真正的平台。
